Cloud Storage / 云存储

Remote Data Storage Technology / 远程数据存储技术

Definition / 定义

  • Method of data storage on remote servers / 在远程服务器上存储数据的方法
  • Data stored on more than one server (data redundancy) / 数据存储在多个服务器上（数据冗余）
  • Allows client access at any time / 允许客户端随时访问
  • Physical environment owned by hosting company / 物理环境由托管公司拥有
  • May include hundreds of servers in many locations / 可能包括多个位置的数百台服务器

Three Cloud Systems / 三种云系统

Public Cloud / 公有云

  • Customer and provider are different companies / 客户和提供商是不同公司的存储环境
  • Customer/client and cloud storage provider are separate entities / 客户/用户和云存储提供商是独立实体
  • Examples: Google Drive, Dropbox, OneDrive / 示例：Google Drive、Dropbox、OneDrive

Private Cloud / 私有云

  • Storage provided by dedicated environment behind company firewall / 公司防火墙后的专用环境提供的存储
  • Customer/client and provider integrated as single entity / 客户/用户和提供商集成为单一实体
  • Greater control and security / 更大的控制和安全性

Hybrid Cloud / 混合云

  • Combination of public and private cloud / 公有云和私有云的组合
  • Some data in private cloud, less sensitive data in public cloud / 部分数据在私有云中, 较不敏感的数据在公有云中
  • Best of both worlds / 两全其美

Benefits & Drawbacks / 优缺点

Anywhere Access / 随处访问

  • Files accessible from any device, anywhere, with an internet connection.

Internet Dependency / 依赖互联网

  • No internet means no access. Slow connections can cause issues.

No Physical Storage / 无需物理存储

  • No need to carry external drives or use the same computer.

Cost Issues / 成本问题

  • High costs for large capacity and expensive data transfer limits.

Remote Backup / 远程备份

  • Automatic data backup offers disaster recovery benefits from hardware failure.

Company Failure Risk / 公司倒闭风险

  • Potential for the cloud company to fail, risking loss of all data.

⚠️
Data Security Issues / 数据安全问题

Relinquishing Control / 放弃控制

  • Transferring confidential data to a provider means giving up direct control over its security.

Physical Security / 物理安全

  • Questions about building security, disaster resistance, and power safeguards.

Personnel Security / 人员安全

  • Risk of provider personnel accessing data or misusing authorization codes.

⚠️
Potential Data Loss & Real Breaches / 潜在数据丢失与真实案例

1. XEN Hypervisor Security Threat / XEN虚拟机管理程序安全威胁

A flaw in the XEN hypervisor forced major cloud operators like Amazon and Rackspace to reboot their entire server fleets, causing widespread service impact.

2. Data Loss During Backup / 备份期间数据丢失

A large cloud provider permanently lost customer data during a routine backup procedure, highlighting the risk of irretrievable data loss even with professionals.

3. Celebrity Photos Cloud Hacking / 名人照片云黑客事件

Hackers gained access to over 100 private celebrity photos by breaching cloud accounts, which were then leaked and sold online.

4. Mexico Electoral Database Breach (2016) / 墨西哥选举数据库泄露(2016)

The registration data of 93 million voters was compromised and made publicly available after being stored on a misconfigured cloud server.

Recommendations / 建议

  • Use strong, unique passwords and enable two-factor authentication (2FA). / 使用强密码和双因素认证
  • Encrypt highly sensitive data *before* uploading it to the cloud. / 上传前加密敏感数据
  • Understand your provider's security policies and terms of service. / 了解提供商的安全政策
  • Maintain local backups of your most critical data (3-2-1 backup rule). / 维护关键数据的本地备份
  • Choose reputable cloud providers with a proven track record of security. / 选择信誉良好的云提供商
  • Conduct regular security audits if using cloud for business purposes. / 定期安全审计
