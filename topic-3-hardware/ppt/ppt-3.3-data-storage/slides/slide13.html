Virtual Memory / 虚拟内存

Memory Management Technique / 内存管理技术

1. Definition / 定义

  • Memory management technique used when RAM runs out / RAM耗尽时的内存管理技术
  • Prevents the system from crashing due to insufficient RAM / 防止因RAM不足导致系统崩溃
  • Virtual Memory = Physical RAM + Swap Space on HDD/SSD / 虚拟内存 = 物理RAM + HDD/SSD上的交换空间
  • Gives programs the illusion of having unlimited memory / 为程序提供拥有无限内存的假象

2. How It Works / 工作原理

When RAM is Full / 当RAM已满时: The OS moves the least recently used data from RAM to a swap file (or page file) on the HDD/SSD to free up RAM for new data. / 操作系统将最近最少使用的数据从RAM移动到HDD/SSD上的交换文件（或页面文件），为新数据释放RAM。

32-bit Address Mapping / 32位地址映射: Programs access memory via a virtual address map. When RAM is full, the OS moves older data to the disk and updates the map to reflect the new location, creating a seamless experience for the program. / 程序通过虚拟地址映射访问内存。当RAM满时，操作系统将旧数据移至磁盘并更新映射以反映新位置，为程序创造无缝体验。

3. Paging / 分页

What is Paging / 什么是分页

  • Used by OS for memory management / 操作系统用于内存管理
  • Stores and retrieves data from secondary storage (HDD/SSD) for use in main memory (RAM) / 从二级存储(HDD/SSD)中存储和检索数据，供主内存(RAM)使用
  • Copies data into fixed-size blocks called pages into RAM as needed / 根据需要将数据以称为页面的固定大小块复制到RAM中

Page Scheduling / 页面调度

  • Determines which pages to keep in RAM and which to move to swap space / 决定哪些页面保留在RAM中，哪些移到交换空间
  • Based on algorithms like LRU (Least Recently Used) / 基于最近最少使用(LRU)等算法

4. Disk Thrashing / 磁盘颠簸

What is Disk Thrashing / 什么是磁盘颠簸

  • Occurs when main memory is full, leading to excessive swapping of data between RAM and virtual memory / 主内存已满时发生，导致RAM和虚拟内存之间数据过度交换
  • Causes a high rate of hard disk read/write operations / 导致硬盘读/写操作频率极高
  • System spends more time swapping data than processing it / 系统花费在交换数据上的时间超过了处理数据的时间

Impact on Performance / 对性能的影响

  • System becomes extremely slow and unresponsive / 系统变得极其缓慢且无响应
  • Can lead to premature hardware failure, especially on HDDs / 可能导致硬件过早失效，尤其是在HDD上

5. Solutions to Reduce Thrashing / 减少颠簸的解决方案

  • Install More RAM / 安装更多RAM: The most effective solution, reducing the need for swapping. / 最有效的解决方案，减少交换需求。
  • Reduce Running Programs / 减少运行程序: Frees up memory by closing unnecessary applications. / 通过关闭不必要的应用程序来释放内存。
  • Use an SSD Instead of an HDD / 使用SSD代替HDD: SSDs have no moving parts and are much faster, significantly mitigating the impact of thrashing. / SSD没有移动部件且速度快得多，可显著减轻颠簸的影响。

6. Benefits of Virtual Memory / 虚拟内存的优点

  • Allows programs to be larger than physical memory / 允许程序大小超过物理内存
  • Enhances multitasking by allowing more programs to run simultaneously / 允许更多程序同时运行，增强多任务处理能力
  • Reduces the need for expensive RAM upgrades / 减少对昂贵RAM升级的需求

7. Drawbacks / 缺点

  • Slower than accessing physical RAM / 访问速度比物理RAM慢
  • Can lead to performance degradation (Disk Thrashing) / 可能导致性能下降（磁盘颠簸）
  • Consumes disk space for the swap file / 消耗磁盘空间用于交换文件
