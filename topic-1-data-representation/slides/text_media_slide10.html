质量与文件大小 / Quality vs File Size

质量提高 → 文件大小增加
Higher quality → Larger file size
存储空间与传输时间的权衡
Balance between storage and transmission
用途决定最佳参数
Usage determines optimal parameters

文本文件 / Text Files

  • ASCII: 1字节/字符，限制语言
    1 byte/char, limited languages
  • Unicode: 1-4字节/字符，支持所有语言
    1-4 bytes/char, all languages

选择原则 / Selection Principle:

根据内容语言需求，现代应用优先使用Unicode (UTF-8) 以保证兼容性。

音频文件 / Audio Files

参数影响 / Parameter Effects:

  • 采样率: 8kHz(电话) → 192kHz(高保真)
    Sample Rate: 8kHz(phone) → 192kHz(hi-fi)
  • 采样分辨率: 8-bit → 32-bit
    Sample Resolution: 8-bit → 32-bit
  • 声道: 单声道 vs 立体声
    Channels: Mono vs Stereo
文件大小 = 采样率 × 分辨率 × 时间 × 声道数 File Size = Sample Rate × Resolution × Time × Channels

图像文件 / Image Files

参数影响 / Parameter Effects:

  • 分辨率: 像素数量决定清晰度
    Resolution: pixel count determines clarity
  • 色彩深度: 1-bit(黑白) → 32-bit(真彩色)
    Color Depth: 1-bit(B&W) → 32-bit(true color)
  • 压缩: 无损 vs 有损
    Compression: lossless vs lossy
文件大小 = 宽度 × 高度 × 色彩深度 File Size = Width × Height × Color Depth

实际应用指南 / Practical Usage Guide

网络传输 / Web Transmission

低分辨率，适度压缩，优先考虑加载速度。
Low resolution, moderate compression, prioritize loading speed.

专业存档 / Professional Archive

高分辨率，无损压缩，保持最高质量。
High resolution, lossless compression, maintain highest quality.

移动设备 / Mobile Devices

中等参数，考虑存储限制，平衡质量和性能。
Medium parameters, consider storage, balance quality & performance.
