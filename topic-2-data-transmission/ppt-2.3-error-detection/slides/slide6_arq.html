自动重复请求

Automatic Repeat Request (ARQ)

1. 核心机制

使用确认和超时机制确保数据正确接收。

Uses acknowledgment and timeout mechanisms to ensure correct data reception.

✅ 2. 正面确认机制 (Positive Acknowledgment)

当数据包正确接收且无错误时，接收方发送ACK信号。

When data packet is correctly received without errors, receiver sends ACK signal.

作用: 通知发送方可以发送下一个数据包 / Notifies sender to send next data packet

❌ 3. 负面确认机制 (Negative Acknowledgment)

当发现数据包错误时，接收方发送NAK信号。

When data packet error is detected, receiver sends NAK signal.

作用: 请求发送方重新传输该数据包 / Requests sender to retransmit that data packet

⏰ 4. 超时机制 (Timeout Mechanism)

发送方设置超时计时器，等待确认信号。

Sender sets timeout timer, waiting for acknowledgment signal.

超过时间限制未收到ACK，自动重传数据包。

If no ACK received within time limit, automatically retransmit data packet.

5. ARQ流程示例

发送方          接收方
  |                |
  |── Packet A ──>|
  |<── ACK A ────|
  |                |
  |── Packet B ──>|
  |<── NAK B ────|  (请求重传)
  |                |
  |── Packet B ──>|  (重传)
  |<── ACK B ────|

优点 (Advantages)

  • 高可靠性，确保数据完整接收
    High reliability, ensures complete data reception
  • 自动错误恢复能力
    Automatic error recovery capability

缺点 (Disadvantages)

  • 网络延迟增加
    Increased network delay
  • 额外的带宽开销
    Additional bandwidth overhead
